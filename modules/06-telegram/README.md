# 06. Telegram

Головний інтерфейс до агента. Ви пишете йому в Telegram-групу з топіками, він відповідає туди ж. Окремі топіки тримають окремі контексти: Вхідні для будь-яких запитів, Пошта для листів, Проєкти для вайбкодингу, Звіти для дайджестів.

## Що ви отримаєте

Свого бота, який працює через long polling: сервер сам ходить до Telegram за повідомленнями, тому жоден порт назовні не відкривається. Кожен топік працює як окрема розмова з агентом і тримає власну пам'ять.

## Що зробити руками

Покроково, займає 5 хвилин. Робити можна і з телефона, і з компʼютера.

### Крок 1. Створіть бота

1. Відкрийте в Telegram чат [@BotFather](https://t.me/BotFather) (перевірте синю галочку: це офіційний бот Telegram).
2. Надішліть команду `/newbot`.
3. BotFather спитає імʼя бота: введіть, наприклад, `Мій агент`.
4. Далі спитає username: він має закінчуватись на `bot` і бути вільним, наприклад `ivan_agent_bot`.
5. У відповідь прийде повідомлення з токеном виду `1234567890:AAF...`. Нікому його не показуйте і нікуди не вставляйте, крім кроку з агентом нижче: цей токен дає повний контроль над ботом.

### Крок 2. Дозвольте боту бачити повідомлення групи

1. У чаті з BotFather надішліть `/setprivacy`.
2. Оберіть зі списку свого бота.
3. Натисніть `Disable`. BotFather підтвердить: privacy mode is disabled.

Без цього кроку бот бачитиме в групі тільки команди, а не звичайні повідомлення.

### Крок 3. Створіть групу з топіками

1. Telegram → нова група. Назва, наприклад, `Мій агент`. Учасник поки один: ви (Telegram може вимагати додати когось: додайте свого бота одразу тут).
2. Відкрийте налаштування групи (тапніть по назві → олівець або Edit).
3. Увімкніть перемикач `Topics` (Теми). Група перетвориться на форум з топіком General.

### Крок 4. Додайте бота адміністратором

1. Налаштування групи → Administrators → Add Admin.
2. Знайдіть свого бота за username і додайте.
3. Залиште стандартні права: головне, щоб були `Manage Topics` і можливість писати.

### Крок 5. Створіть чотири топіки

1. У групі натисніть на список тем → кнопка створення нового топіка (олівець або `+`).
2. Створіть по черзі чотири топіки з точними назвами: `Вхідні`, `Пошта`, `Проєкти`, `Звіти`.

Назви важливі: агент розкладає повідомлення саме за ними. Топік General можна лишити або приховати.

## Що зробить агент

На сервері запустіть `claude` і вставте цілком файл
[prompts/telegram-bridge.md](prompts/telegram-bridge.md). Агент попросить токен бота з кроку 1 (збереже його в 1Password), напише міст між Telegram і Claude Code та поставить його як системний сервіс, який переживає перезавантаження сервера.

## Перевірка

1. Напишіть у топік Вхідні: «Привіт, хто ти?». Протягом хвилини бот відповість.
2. Перезавантажте сервер (`sudo reboot`), зачекайте дві хвилини і напишіть боту ще раз. Відповідає: сервіс піднявся сам.

## Якщо щось не працює

- Бот мовчить у групі, але відповідає в особистих: не виконаний крок 2 (`/setprivacy` → Disable) або бот не адмін.
- Бот відповідає не в той топік: перевірте точні назви топіків з кроку 5.
- Бот мовчить всюди: на сервері виконайте `systemctl status agent-bridge` або просто спитайте агента через SSH, що з мостом.

## Куди зберегти доступи

- "Telegram Bot Token (agent)": створить агент через op під час налаштування.
