# 06. Telegram

Головний інтерфейс до агента. Ви пишете йому в Telegram-групу з топіками, він відповідає туди ж. Окремі топіки тримають окремі контексти: Вхідні для будь-яких запитів, Пошта для листів, Проєкти для вайбкодингу, Звіти для дайджестів.

## Що ви отримаєте

Свого бота, який працює через long polling: сервер сам ходить до Telegram за повідомленнями, тому жоден порт назовні не відкривається. Кожен топік це окрема розмова з агентом зі своєю пам'яттю.

## Що зробити руками

1. У Telegram відкрийте [@BotFather](https://t.me/BotFather) → `/newbot` → придумайте ім'я і username бота. Скопіюйте токен.
2. У BotFather: `/setprivacy` → оберіть свого бота → Disable (щоб бот бачив повідомлення в групі).
3. Створіть нову групу, назвіть її, наприклад, "Мій агент". У налаштуваннях групи увімкніть Topics.
4. Додайте бота в групу і зробіть його адміністратором (потрібно для роботи з топіками).
5. Створіть чотири топіки: `Вхідні`, `Пошта`, `Проєкти`, `Звіти`.

## Що зробить агент

На сервері запустіть `claude` і вставте цілком файл
[prompts/telegram-bridge.md](prompts/telegram-bridge.md). Агент попросить токен бота (збереже його в 1Password), напише міст між Telegram і Claude Code та поставить його як системний сервіс, який переживає перезавантаження сервера.

## Перевірка

1. Напишіть у топік Вхідні: "Привіт, хто ти?". Протягом хвилини бот відповість.
2. Перезавантажте сервер (`sudo reboot`), зачекайте дві хвилини і напишіть боту ще раз. Відповідає: сервіс піднявся сам.

## Куди зберегти доступи

- "Telegram Bot Token (agent)": створить агент через op під час налаштування.
