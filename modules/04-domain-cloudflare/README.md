# 04. Домен і Cloudflare

Ваш домен переїжджає під керування Cloudflare. Він потрібен не для агента (агент назавжди лишається без публічної адреси), а для ваших майбутніх проєктів з модуля 11: кожен отримає свій субдомен однією командою.

## Що ви отримаєте

Домен на Cloudflare і токен, яким агент сам створює субдомени для ваших проєктів. Правило безпеки: для сервера агента DNS-записів не існує.

## Що зробити руками

1. Якщо домену ще немає: купіть його. Найпростіше прямо в Cloudflare: Dash → Domain Registration → Register domain. Підійде і будь-який інший реєстратор.
2. Якщо домен куплено не в Cloudflare: у [dash.cloudflare.com](https://dash.cloudflare.com) → Add a domain → введіть домен → оберіть план Free → Cloudflare покаже два NS-сервери → у панелі свого реєстратора замініть NS-записи на ці два. Зміна NS може тривати до доби.
3. Створіть API-токен для агента: My Profile → API Tokens → Create Token → шаблон Edit zone DNS.
   - Zone Resources: Include → Specific zone → ваш домен.
   - Permissions: Zone / DNS / Edit і Zone / Zone / Read.
4. Скопіюйте токен: він знадобиться агенту.

## Що зробить агент

На сервері запустіть `claude` і вставте цілком файл
[prompts/cf-subdomain.md](prompts/cf-subdomain.md). Першим запуском агент збереже токен у 1Password і перевірить доступ до зони. Далі цей самий промпт використовується щоразу, коли треба створити субдомен для нового проєкту.

## Перевірка

Агент наприкінці показує список DNS-записів зони. Домен у Cloudflare має статус Active.

## Куди зберегти доступи

- "Cloudflare DNS token (домен)": створить агент через op. Перевірте, що айтем з'явився у сейфі Agent.
