# 03. 1Password

Всі доступи, які створюватиме агент (токени, паролі, ключі), зберігаються в одному сейфі 1Password. Ви бачите все, агент дістає секрети без вашої участі, і жоден секрет не лежить у файлах відкритим текстом.

## Що ви отримаєте

Окремий сейф Agent у вашому 1Password. Агент вміє класти туди нові доступи і читати їх через сервісний акаунт з доступом тільки до цього сейфа.

## Що зробити руками

1. У застосунку 1Password: New vault → назва `Agent`.
2. Створіть сервісний акаунт: [my.1password.com/developer-tools/infrastructure-secrets/serviceaccount](https://my.1password.com/developer-tools/infrastructure-secrets/serviceaccount) → Create service account.
   - Name: `agent-vps`.
   - Vault access: тільки сейф `Agent`, права read і write.
3. Скопіюйте токен сервісного акаунта (показується один раз). Тримайте його під рукою для наступного кроку.

## Що зробить агент

На сервері запустіть `claude` і вставте цілком файл
[prompts/op-bootstrap.md](prompts/op-bootstrap.md). Агент попросить вставити токен, встановить 1Password CLI і перевірить доступ до сейфа.

## Перевірка

На сервері:

```sh
op vault list
```

У списку рівно один сейф: Agent.

## Куди зберегти доступи

- "1Password service account (agent-vps)": сам токен сервісного акаунта. Покладіть його в сейф Agent руками: це єдиний секрет, який агент не може зберегти сам.
