# 08. Obsidian

Ваш другий мозок. Нотатки живуть простими файлами у вас, синхронізуються між телефоном і ноутбуком через Obsidian Sync, а агент бачить їх на сервері через git: читає, доповнює, веде щоденні записи. Ведеться все за методом Карпати: спека перед справою, маленькі кроки, рішення записуються і звʼязуються.

## Що ви отримаєте

- Сховище нотаток Brain на всіх ваших пристроях: телефон, ноутбук, сервер агента.
- Все, що ви кинете у нотатку "0-inbox", агент розбере і розкладе.
- Ранковий дайджест агент дописує у щоденну нотатку: історія ваших днів збирається сама.
- Метод роботи, за якого нотатки стають памʼяттю компанії: кожен проєкт починається зі спеки, кожне рішення лишає слід.

## Дві синхронізації, кожна для свого

| Канал                             | Для чого                                                        | Ціна            |
| --------------------------------- | --------------------------------------------------------------- | --------------- |
| Obsidian Sync (офіційна підписка) | миттєва синхронізація телефон ↔ ноутбук, шифрування end-to-end  | Standard $4/міс |
| git-репо на GitHub                | канал агента (сервер бачить нотатки) + версійна історія + бекап | безкоштовно     |

Можна жити і на самому git, але Sync дає нотатки на телефоні без жодних налаштувань, тому кіт рекомендує обидва.

## Що зробити руками

1. Встановіть [Obsidian](https://obsidian.md/download) на компʼютер і телефон.
2. Створіть нове сховище (vault) з назвою `Brain`.
3. Оформіть підписку [Obsidian Sync](https://obsidian.md/sync) (Standard, $4/міс): Settings → Sync → Set up → створіть remote vault `Brain` → підключіть телефон до того самого remote vault. Пароль шифрування збережіть у 1Password.
4. Створіть приватне репо на GitHub: [github.com/new](https://github.com/new) → назва `brain` → Private → Create.
5. В Obsidian на компʼютері встановіть плагін Git: Settings → Community plugins → Turn on → Browse → знайдіть `Git` (obsidian-git) → Install → Enable.
6. Згенеруйте GitHub-токен для плагіна: [github.com/settings/personal-access-tokens](https://github.com/settings/personal-access-tokens) → Fine-grained token → доступ тільки до репо `brain`, права Contents: Read and write. Збережіть токен у 1Password.
7. Підключіть сховище до репо за інструкцією плагіна obsidian-git (вставте URL репо з токеном).

## Що зробить агент

Два промпти, за порядком:

1. [prompts/vault-sync.md](prompts/vault-sync.md): склонує сховище на сервер, налаштує синхронізацію кожні 15 хвилин і створить стартову структуру.
2. [prompts/vault-method.md](prompts/vault-method.md): налаштує метод Карпати у сховищі: теку спек, шаблон рішення, конвенції атомарних нотаток, і навчить агента ними користуватись.
3. Опційно, якщо ваші зустрічі пише Fathom: [prompts/fathom-meetings.md](prompts/fathom-meetings.md): кожна записана зустріч автоматично зберігається у сховище як нотатка з резюме, рішеннями і транскриптом, групується по людях і проєктах через лінки, а коротке резюме прилітає в Telegram.

## Метод Карпати у двох словах

Як цим користуватись щодня:

- **Спека перед справою.** Новий проєкт або велика задача починається не з роботи, а з короткої нотатки-спеки: що робимо, навіщо, як зрозуміти, що вийшло. Кажете агенту в Telegram «нова спека: виходимо на ринок Польщі», він створює нотатку з шаблону і ставить питання.
- **Маленькі перевірювані кроки.** Агент розбиває ціль зі спеки на кроки, кожен з яких можна перевірити окремо. Прогрес відмічається у тій самій нотатці.
- **Рішення записуються.** Кожне важливе рішення отримує датовану нотатку: контекст, варіанти, чому обрали це. Через 90 днів агент сам нагадає перевірити, чи справдились припущення.
- **Все звʼязане.** Нотатки посилаються одна на одну через [[подвійні дужки]]: спека на рішення, рішення на людей і проєкти. Через пів року це працює як памʼять компанії, у якій агент вміє шукати.

## Перевірка

1. Створіть нотатку на телефоні: за хвилину вона зʼявилась на компʼютері (Sync працює).
2. Напишіть агенту в Telegram: «що написано в моїй останній нотатці?». Він відповість текстом нотатки (git-канал працює).
3. Напишіть агенту: «нова спека: тест методу». У теці specs зʼявилась нотатка за шаблоном.

## Куди зберегти доступи

- "Obsidian Sync password (Brain)": пароль шифрування remote vault, покладіть руками.
- "GitHub token (brain repo)": fine-grained токен з кроку 6, покладіть руками.
