# 07. Пошта

Агент бере на себе дві ваші скриньки Gmail: читає нове, розкладає по ярликах, зіркою позначає важливе, термінове одразу шле в Telegram, а щоранку присилає дайджест. Ви відкриваєте пошту вже відсортованою.

## Що ви отримаєте

- Ярлики в обох скриньках: Важливе, Потребує відповіді, Рахунки, Шум.
- Термінові листи (гроші, дедлайни, VIP-люди з вашого профілю) прилітають у топік Пошта одразу.
- Щоранковий дайджест: хто написав, про що, чому це важливо, які дедлайни.

## Що зробити руками

1. Створіть проєкт у Google Cloud: [console.cloud.google.com](https://console.cloud.google.com) → New Project → назва `agent-mail`.
2. Увімкніть Gmail API: APIs & Services → Library → Gmail API → Enable.
3. Налаштуйте OAuth-екран: APIs & Services → OAuth consent screen → External → заповніть назву (`agent`) і свою пошту → у Test users додайте **обидві** ваші адреси.
4. Створіть OAuth-клієнт: APIs & Services → Credentials → Create Credentials → OAuth client ID → Desktop app. Скопіюйте Client ID і Client Secret.
5. Тримайте обидві скриньки під рукою: під час налаштування агент дасть два посилання для входу, по одному на кожну.

## Що зробить агент

На сервері запустіть `claude` і вставте цілком файл
[prompts/mail-triage.md](prompts/mail-triage.md). Агент збереже OAuth-доступи в 1Password, проведе вас через авторизацію обох скриньок, створить ярлики і запустить перший розбір пошти.

## Перевірка

1. В обох скриньках з'явились ярлики Agent/Важливе, Agent/Відповісти, Agent/Рахунки, Agent/Шум.
2. Надішліть собі тестовий лист зі словами "терміново, дедлайн завтра" з чужої адреси: протягом години (або після ручного запуску розбору) він з'явиться у топіку Пошта.

## Куди зберегти доступи

- "Gmail OAuth client (agent-mail)": Client ID і Secret, створить агент через op.
- "Gmail token (адреса 1)" і "Gmail token (адреса 2)": refresh-токени обох скриньок, теж через op.
