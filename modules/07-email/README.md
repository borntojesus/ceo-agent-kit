# 07. Пошта: тріаж, сортування, управління

Агент бере на себе дві ваші скриньки Gmail повністю: сортує кожен новий лист по папках, підсвічує важливе, тримає інбокс порожнім, готує чернетки відповідей і слухається команд у Telegram. Ви відкриваєте пошту вже розібраною, а більшість листів взагалі не бачите: вони чекають у своїх папках або у дайджесті.

## Що ви отримаєте

**Сортування.** Сім папок у кожній скриньці, кожен новий лист потрапляє рівно в одну:

| Папка            | Що туди йде                     | Що з цим робить агент                                |
| ---------------- | ------------------------------- | ---------------------------------------------------- |
| Agent/Важливе    | VIP-люди, гроші, дедлайни       | зірка + миттєве повідомлення в Telegram              |
| Agent/Відповісти | прямі питання до вас            | чернетка відповіді вже чекає в Drafts                |
| Agent/Рахунки    | інвойси, чеки, платежі          | сума й термін у дайджесті, PDF у теку для бухгалтера |
| Agent/Розсилки   | ньюслетери, дайджести           | тижневий огляд одним листом у неділю                 |
| Agent/Сповіщення | сервісні нотифікації, соцмережі | архів одразу, без згадок                             |
| Agent/Холодне    | cold outreach, пропозиції       | короткий список раз на тиждень                       |
| Agent/Шум        | все інше                        | архів                                                |

**Управління.** Інбокс тримається порожнім: розібраний лист архівується, важливе живе зіркою і папкою. Раз на місяць агент показує список розсилок, які ви не відкривали, і відписує від зайвих після вашого підтвердження.

**Команди в Telegram** (топік Пошта):

- «що нового»: зведення непрочитаного по обох скриньках
- «драфт відповіді на лист від Олени»: чернетка у Drafts, текст у Telegram
- «знайди лист про оренду офісу»: пошук по обох скриньках
- «відпишись від цієї розсилки»: відписка через службовий заголовок
- «додай domain.com у VIP»: правило сортування оновлюється одразу

## Що зробити руками

1. Створіть проєкт у Google Cloud: [console.cloud.google.com](https://console.cloud.google.com) → New Project → назва `agent-mail`.
2. Увімкніть Gmail API: APIs & Services → Library → Gmail API → Enable.
3. Налаштуйте OAuth-екран: APIs & Services → OAuth consent screen → External → заповніть назву (`agent`) і свою пошту → у Test users додайте **обидві** ваші адреси.
4. Створіть OAuth-клієнт: APIs & Services → Credentials → Create Credentials → OAuth client ID → Desktop app. Скопіюйте Client ID і Client Secret.
5. Тримайте обидві скриньки під рукою: під час налаштування агент дасть два посилання для входу, по одному на кожну.

## Що зробить агент

Два промпти, за порядком:

1. [prompts/mail-triage.md](prompts/mail-triage.md): підключить обидві скриньки, створить сім папок, навчиться вашим правилам сортування і прожене перший розбір.
2. [prompts/mail-manage.md](prompts/mail-manage.md): додасть управління: порожній інбокс, чернетки відповідей, тижневий огляд розсилок, відписки, команди в Telegram.

## Правила безпеки, вшиті в модуль

- Агент **ніколи не відправляє листи сам**: тільки чернетки у Drafts. Кнопку Send завжди натискаєте ви.
- Агент **нічого не видаляє**: лише архівує. Будь-який лист можна знайти пошуком.
- Відписки тільки через службовий заголовок List-Unsubscribe і тільки після вашого «так» у Telegram. За посиланнями «unsubscribe» у тілі листа агент не ходить: серед них бувають пастки.

## Перевірка

1. В обох скриньках з'явились сім папок Agent/\*.
2. Надішліть собі тестовий лист зі словами «терміново, дедлайн завтра» з чужої адреси: він прилетить у топік Пошта.
3. Напишіть у топік Пошта «що нового»: агент відповість зведенням.
4. Вранці наступного дня інбокс порожній, все розібране по папках.

## Куди зберегти доступи

- "Gmail OAuth client (agent-mail)": Client ID і Secret, створить агент через op.
- "Gmail token (адреса 1)" і "Gmail token (адреса 2)": refresh-токени обох скриньок, теж через op.
